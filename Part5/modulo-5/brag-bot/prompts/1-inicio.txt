Spec 02: Unipds Visual Interface and Data Mocking
Context: Angular 21 project with Tailwind CSS. Genkit and MCP are already installed.

Interface and Routing Objectives:
1 - Visit https://unipds.com.br/org-pos-ia/ and extract the visual identity.
2 - Create two standalone components: 'DashboardComponent' and 'DetailComponent'. Configure 'app.routes.ts' to navigate between them (default route for dashboard and '/detail/:id' for the detail view).
3 - Create a 'BragService' using Angular Signals. It must include:
- a "brags" signal storing an array of achievements
- a "loading" signal (boolean)
- a 'generateMockBrag(prompt: string)' method that simulates a network call (e.g., a 1.5s 'setTimeout'), activates the loading state, and then adds a static mocked JSON object (with Title, Context, Impact, and Technologies) to the "brags" list.
4 - In DashboardComponent (HTML):
- create a header with the title "Brag-bot | Pos IA Unipds"
- create a form with a '<textarea>' for the raw achievement and a "Distill Achievement" button. The button should show a spinner or change text while 'loading' is true.
- Below the form, use Angular's '@for' loop to render cards for the generated achievements. Each card must be clickable, redirecting to the DetailComponent.
5 - In DetailComponent (HTML):
- render the full achievement details (Context, Impact, Metrics, and Technology) with a reader-friendly layout. Add a "Back to Dashboard" button.
