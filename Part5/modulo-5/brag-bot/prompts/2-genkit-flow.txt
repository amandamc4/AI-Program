Spec 03: Creating a Genkit Flow via MCP (BragBot)

Prerequisite instruction: USE THE GENKIT MCP. Consult the connected MCP context to use the correct Genkit API syntax.

Act as a Senior Software Engineer specializing in creating AI-based applications using the Firebase Genkit framework.

Your task is to create a TypeScript file (e.g., 'src/flows.ts') that defines data schemas and a Genkit flow to transform a user's informal notes about their work achievements into a professional, well-structured "Brag Document."

Technical and configuration specifications:
1 - Import 'genkit' and 'z' from 'genkit'.
2 - Import the Google AI plugin: 'googleAI' from '@genkit-ai/google-gen-ai'.
3 - Import the 'v4' function from the 'uuid' library as 'uuidv4'.
4 - Initialize the Genkit instance, configuring it to use the Google AI plugin and setting 'gemini-flash-latest' as the default model with a temperature of 0.8. Schema Specifications (use Zod and add `.describe()` to all fields to guide the LLM):
1 - BragInputSchema: must contain a 'definition' field (string), representing the user's informal draft.
2 - BragSchema (the strict JSON output schema):
- title: string (e.g., Main action + high-level result)
- context: string (original situation/problem; what was broken, slow, etc.)
- actionTaken: string (step-by-step technical or strategic action taken to solve the problem)
- businessImpact: string (business impact; time saved, failure reduction, etc.)
- metrics: array of strings (strictly quantifiable data only; e.g., "50% reduction")
- technologiesUsed: array of strings (tools, languages, and platforms mentioned or inferred)

Flow Implementation
1 - Create and export a flow named 'bragGeneratorFlow' using `ai.defineFlow`.
2 - Configure it to accept BragInputSchema and return BragSchema.
3 - Within the flow body, construct a prompt string with the following guidelines:
- Persona: The model must act as a Senior Career Consultant focused on Individual Development Plans (IDP) for software engineers.
- Objective: Transform the user's informal draft into an executive-level Brag Document.
- Rule 1: Use a professional, objective, and impact-focused tone; avoid emotional adjectives.
- Rule 2: If exact metrics are missing, the AI ​​must infer the nature of the metric based on the action taken.
- Rule 3: Strictly follow the BragSchema JSON format.
- Rule 4: The output must match the input's language (if submitted in Portuguese, respond in Portuguese).
4 - Include `input.definition` at the end of the prompt. 5 - Call the Genkit content generation function (ai.generate), passing in the constructed prompt and requiring the output to adhere to BragSchema.
6 - Check if a valid output exists; if not, throw an error.
7 - The function's final return value must include all properties mapped by the AI, plus an 'id' field containing a natively generated UUID (uuidv4()).
